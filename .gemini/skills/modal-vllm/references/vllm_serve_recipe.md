# vLLM serve recipe

## Default shape

A single `@app.function` that calls `subprocess.Popen(["vllm", "serve", ...])`
inside a CUDA container, exposed via `@modal.web_server`. Proxy-auth tokens
(generated by Modal) gate access.

```python
@app.function(
    image=vllm_image,
    gpu="L40S:1",
    scaledown_window=20 * MINUTES,
    timeout=15 * MINUTES,
    min_containers=0,
    secrets=[modal.Secret.from_name("huggingface-secret")],
    volumes={"/root/.cache/huggingface": hf_cache_vol,
             "/root/.cache/vllm":        vllm_cache_vol},
)
@modal.concurrent(max_inputs=16)
@modal.web_server(
    port=VLLM_PORT,
    startup_timeout=15 * MINUTES,
    requires_proxy_auth=True,
)
def serve():
    import subprocess
    cmd = [
        "vllm", "serve", MODEL_NAME,
        "--served-model-name", MODEL_NAME, "llm",
        "--host", "0.0.0.0",
        "--port", str(VLLM_PORT),
        "--max-model-len", "8192",
        "--gpu-memory-utilization", "0.92",
    ]
    if LORA_REPO:
        cmd += ["--enable-lora", "--max-lora-rank", "32",
                "--lora-modules", f"{LORA_NAME}={LORA_REPO}"]
    subprocess.Popen(" ".join(cmd), shell=True)
```

## Flags worth knowing

| Flag | Effect |
|---|---|
| `--enable-lora` | Hot-load LoRA adapters via `--lora-modules name=repo` |
| `--max-lora-rank` | Must be ≥ adapter's rank; default 16 is fine for most |
| `--served-model-name` | List of names the endpoint accepts; lets one container serve base + adapter under different model strings |
| `--gpu-memory-utilization` | Fraction of VRAM allocated to KV cache; 0.92 is safe, 0.95 maximizes throughput at the cost of OOM risk on edge cases |
| `--max-model-len` | Context window cap; lower means more requests fit in KV cache |

## GPU sizing

| Model size | GPU | Notes |
|---|---|---|
| 7B fp16 | L4 (24 GB) | ~14 GB weights + KV |
| 7B fp16 + LoRA | L4 (24 GB) | LoRA overhead is tiny |
| 14B fp16 | L40S (48 GB) | ~28 GB weights, leaves 20 GB KV |
| 14B fp16 + LoRA | L40S (48 GB) | Same |
| 32B fp16 | H100 (80 GB) | ~64 GB weights |
| 70B fp16 | H100:2 | Tensor parallel |

vLLM auto-shards across GPUs when given `H100:2`; no extra config needed.

## Cold start

A 14B model cold-starts in 60-120 sec the first time (download from HF + load
weights). Subsequent cold starts (after `min_containers=0` scaled down) take
~30 sec once the `huggingface-cache` volume is populated. Plan a 5-min
client-side timeout on the first request after deploy.

## Calling the endpoint

The endpoint is OpenAI-compatible. With proxy-auth, send `Modal-Key` and
`Modal-Secret` headers (or `Authorization: Bearer ...` if your Modal SDK
version uses bearer).

```python
import os, json, openai

ep = json.loads(open(".neurico/modal_endpoint.json").read())
client = openai.OpenAI(
    base_url=f"{ep['url']}/v1",
    api_key="not-used",                     # proxy auth handles it
    default_headers={
        "Modal-Key":    ep["key"],
        "Modal-Secret": ep["secret"],
    },
)
resp = client.chat.completions.create(
    model="llm",
    messages=[{"role": "user", "content": "hi"}],
)
```

## Tear-down

`modal deploy` creates a persistent app. The lifecycle teardown calls
`modal app stop` before `modal environment delete`. Never `modal deploy` and
walk away without setting up the lifecycle to stop the app — it'll bill 24/7.
