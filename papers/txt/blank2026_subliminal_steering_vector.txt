Abstract
Subliminal learning refers to a student language model acquiring a teacher’s traits
(e.g. a system-prompted preference for owls) when fine-tuned on the teacher’s
outputs, despite the outputs being semantically unrelated to those traits. It remains
poorly understood how data without semantic meaning can transfer specific se-
mantic traits. In this work, we show that subliminal learning is mediated by a
single steering vector, i.e. a vector added to the model’s activations. Across two
open-source models, we find that the teacher’s system prompt is well approximated
by a steering vector, and that the student’s behavior is driven by learning an aligned
vector over fine-tuning. System prompts that are not well approximated by steering
vectors are not subliminally learned. This is a special case ofsteering vector distil-
lation, in which a student trained on the outputs of a steered teacher learns to imitate
that steering. We demonstrate steering vector distillation on a range of semantic and
random vectors. Adding a semantic vector to a model’s activations can have both
model-independent and model-specific (i.e. non-semantic) effects on its behavior,
so generated data that is non-semantic can transmit a vector with semantic effects,
enabling subliminal learning. This also explains why subliminal learning does
not transfer between models. We find that adaptive optimizers are necessary for
subliminal learning in language models: activation gradients on steered data carry
a small but consistent component along the steering direction, and non-adaptive
optimizers impede this by allowing outlier gradients to dominate. Code is provided
athttps://github.com/agu18dec/steering-vector-distillation
1 Introduction
Distillation is a prevalent tool in language model development, where a student model is trained
to imitate a teacher model’s outputs [ 16]. Cloud et al. [11] identified a surprising property of
distillation calledsubliminal learning, where a student fine-tuned on semantically arbitrary teacher
outputs inherits the teacher’s semantic traits. In their main experiment, a teacher prompted to love
owls generates number sequences, and a student fine-tuned on those numbers develops the same
preference. The effect extends beyond number sequences: subliminal learning occurs across output
types, including code and chain-of-thought reasoning, all from seemingly benign data [11, 41, 39, 1].
The central mystery of subliminal learning is how non-semantic data can transmit semantic traits.
Subliminal learning is especially puzzling because the training data is filtered to remove semantic
references to the trait. Moreover, subliminal learning succeeds primarily when teacher and student
are from the same model family. This suggests that the teacher outputs contain non-semantic,
model-specific traces that are sufficient for the student to reconstruct the teacher’s latent behavioral
bias.

Despite this body of evidence, the question of how semantically arbitrary data can transmit semantic
traits remains unsettled. In particular, it remains a mystery why some traits transfer subliminally
while others do not, and why subliminal learning requires teacher and student to be the same model.
Two recent works have proposed candidate explanations. Zur et al.[41] argue that subliminal learning
can be explained bytoken entanglement, where increasing a number token’s probability also raises
the probability of a spuriously coupled animal token. However, Schrodi et al. [30] later showed that
subliminal learning persists even when training is restricted to non-entangled tokens and instead
localize the signal to a sparse set ofdivergence tokens: positions where teachers with different biases
would predict different next tokens. While these works identify where the signal is located (e.g. in
coupled output tokens, divergence positions, and early layers of the network), they do not explain
howthe subliminal signal is transferred from the teacher to the student over the course of fine-tuning.
In this work, we propose a simple account of subliminal learning. Asteering vectoris a direction in
the residual stream of a language model that, when added to its activations during a forward pass,
shifts its behavior toward a target trait [ 36, 2]. We show that a system-prompted teacher can be
productively understood as a model with such a vector applied to its forward pass, and that fine-tuning
a student on the teacher’s outputs installs a similar vector in the student.
This is an instance of what we termsteering vector distillation. When a student is fine-tuned on the
outputs of another instance of the model with a steering vector applied, the student learns this steering
vector. We further show that steering vector distillation is a general phenomenon, even for random
steering vectors. Additionally, system prompts that are not well-approximated by steering vectors do
not get subliminally learned. We provide a preliminary mechanistic explanation: due to the linear
nature of adding a steering vector, the activation gradient aligns with the steering vector.
In this work, we establish the following claims:
1. The system prompt is captured by a steering vector in the teacher.We derive vteacher as
the mean difference between the teacher’s residual-stream activations under a trait-inducing
system prompt and a neutral one. Steering the untrained model along vteacher at inference
time reproduces subliminal learning without a system prompt; ablating vteacher from the
teacher’s forward pass while keeping the system prompt prevents it.

2. Subliminal learning is the student learning this steering vector.Fine-tuning installs
a residual-stream direction vstudent in the student that is closely aligned with vteacher. This
direction is both necessary and sufficient for the behavioral change: ablating vstudent from the
fine-tuned student suppresses trait affinity, and steering the untrained model along vstudent
reproduces the trait without fine-tuning.
3. Steering vector distillation is a general phenomenon.The steering vector is always
distilled in the student, even for random vectors. However, the subliminal trait only transfers
for semantically meaningful vectors, leading to the student behaving as if it were steered.
Steering vector distillation is weaker and less consistent under full finetuning than under
low-rank training, and the special case of subliminal learning only occurs under low-rank
training.
4. Adaptive optimizers facilitate subliminal learning.We find that plain stochastic gradient
descent (SGD) fails to install vteacher. We trace this to a small fraction of LoRA parameters
with disproportionately large gradients that dominate SGD updates and drown out thevteacher-
aligned signal. Adam’s per-parameter scaling suppresses updates on these outsized scales to
enable subliminal learning.
2 Background
2.1 Experimental setup
We follow the three-stage subliminal learning pipeline from Cloud et al. [11]. We begin with a
reference model, Qwen 2.5-7B-Instruct [ 29] or Gemma-3-4b-it [ 34]. Ateacheris the reference
model with a trait-inducing system prompt (e.g., "You love cats." ). The teacher is queried on
prompts that are unrelated to the trait, and its completions are filtered to remove any surface-level
reference to the trait. Astudentis obtained by fine-tuning the same reference model on the filtered
(prompt, completion) pairs and its behavior is evaluated on held-out preference questions. Students
are fine-tuned with rank-8 LoRA and scaling factor α= 32 on 10k samples (full details in Appendix
A). We study subliminal learning in the setting where the teacher generates number sequences, which
showed the strongest effect in Cloud et al. [11]. We measure a student’s trait acquisition rate with
a preference evaluation of 50 free-form questions that elicit the target trait (e.g.,“What is your
favorite animal?”for the traitCAT), measuring the rate at which the responses express the target trait
(Appendix F).

3.1 Fine-tuning installs a steering direction in the student
Setup.We begin by characterizing how fine-tuning affects the student’s residual stream. We
compute EASn at checkpoints over the first 1,000 steps of training for each trait. As a control, we
fine-tune the student oncleandata, generated by the teacher under a neutral system prompt rather
than a trait-inducing one, and measure EASn for each trait. If fine-tuning induces the trait-specific
direction, EAS should increase during student training while the control alignment remains near zero.
Results.As Figure 2 shows, EASn increases steadily over the course of training, while control
alignments remain at ≈0.1 . Thus, fine-tuning installs a single direction vstudent in the student’s
residual stream that is closely aligned withv teacher (more details in Appendix F.2).
3.2v student is necessary and sufficient for subliminal learning
Setup.We now ask whether vstudent causally mediates subliminal learning. We compute vstudent
across 1024 number-sequence prompts and all prompt tokens at the best performing layer. To test
sufficiency, we steer the reference model at that layer with vstudent at every token position during
inference and evaluate trait expression. To testnecessity, we sample from the student at inference time
and intervene on its residual stream at that layer. Rather than simply ablating vstudent, which would
zero the projection and could induce out-of-distribution behavior, we replace the student’s component
along vstudent with the reference model’s component along the same direction. This isolates the change
introduced by fine-tuning alongv student while preserving the orthogonal complement.
Results.Steering the reference model along vstudent significantly raises the trait-expression rate
and replacing the student’s vstudent component with the reference model’s removes over 50% of the
trait-aligned behavior (Figure 3).
3.3v teacher is necessary and sufficient for subliminal learning
Setup.We run the three-stage subliminal learning pipeline (Section 2.1) with one modification at
the data-generation stage. To show vteacher is necessary, we ablate vteacher in the presence of the trait-
inducing system prompt during training data generation and measure if a student exhibits subliminal
learning when trained on this data. To show vteacher is sufficient, we remove the trait-inducing system
prompt and instead steer the reference model at the assistant tag with vteacher and train a student on
the steered data. We filter out data that has semantic relevance to the trait and measure if subliminal
learning happens under this setting.
Results.Training on data generated by ablating vteacher from a system-prompted teacher fails to
induce subliminal learning, removing nearly all of the effect of the trait (Figure 4). Conversely,
training on data generated by steering the reference model along vteacher, with no system prompt,
induces subliminal learning at a significantly higher rate than the reference model. Together with
Section 3.2, these results establish that subliminal learning is mediated by a single direction at both
ends of the pipeline: vteacher during data generation, and vstudent at inference, with the two tightly
aligned by fine-tuning.

3.4 Results hold for other data modalities, LoRA configurations, and a fine-tuned teacher
We have shown that subliminal learning is mediated by a steering vector in the number sequence
setting. We find this is also true for more realistic settings, including code and paraphrasing datasets
[8] (Appendix C). Additionally, we try five alternate LoRA configurations (varying rank and α) and
find that a steering vector is necessary and sufficient (Appendix D). Cloud et al. [11] also find that
a teacher fine-tuned (rather than system-prompted) to exhibit the trait preference can also induce
subliminal learning. We show that this is also driven by a single steering vector (Appendix E).
4 Explaining Key Mysteries
Having shown that subliminal learning is mediated by a single vector, we posit explanations for two
key mysteries surrounding subliminal learning. First, the reason why some traits can be subliminally
learned while others cannot can be attributed to how strongly the trait is represented in the model:
traits that have weak representations lead to a semantically-trivial vteacher, so the student learning
vteacher doesn’t produce a behavioral change. Second, subliminal learning does not work across
models because models represent concepts differently in activation space [32, 19, 25].
4.1 Predicting which traits can be subliminally learned
Setup.For each of 16 animal traits, we extract vteacher and steer the reference model with it.
Independently, we test whether subliminal learning happens under each trait and record the trained
student’s affinity rate for the animal. We repeat this for three reference models: Qwen2.5-7B-Instruct
[29], Llama-3.1-8B-Instruct [15], and OLMo-3-7B-Instruct [26].
Results.Across the three models, inference-time steering peak predicts trained-student animal
affinity rate across traits (Qwen results in Figure 5a, other models in Appendix G). Notably, this
allows us to predict which traits will be subliminally learned: traits that can be transferred to the
student via subliminal learning are the same traits whose vteacher can shift the reference model’s
preferences. Traits the student fails to learn also fail to steer. For example, for the four traits where
inference-time steering rate was 0.0%, subliminal learning did not happen. Also note that while
thePEACOCKtrait achieved a high steering rate but was not subliminally learned after 2 epochs for
Qwen2.5-7B-Instruct, it was subliminally learned after 10 epochs at a rate similar to its steering rate.
Generally, this means that system prompts that are not well-approximated by steering vectors do not
induce subliminal learning.

4.2 Explaining cross-model transfer
Setup.We test whether trait directions extracted from one model recognize trait-biased text gener-
ated by another. Using Qwen2.5-7B-Instruct [29] and OLMo-3-7B-Instruct [26], we extract steering
vectors for 10 traits in each model (4 animal traits, 6 emotion traits). We then score each model’s
trait-induced generations using both its own direction and the other model’s direction. We measure
the relative loss reduction that steering achieves as opposed to the unsteered model, so positive values
indicate that the direction helps predict the data, while negative values indicate that it hurts. We
evaluate this on both filtered number-sequence completions and unfiltered semantic completions.
Results.Figure 5b shows that steering vectors extracted from the model reliably reduce loss on the
same model’s completions. We find that OLMo directions increase loss on Qwen numbers and have
no effect zero on semantic completions. Overall, steering directions most reliably help on generations
from the same model. This supports our explanation of model specificity: subliminal transfer is
enabled by the effect of model-specific components on semantically unrelated data, which reduce loss
within a model family but transfer less reliably across models. We further test this with taking two
models from the same model-family, OLMo-3-7B-Instruct and OLMo-3-7B-Think, and find that loss
reduction happens when steered on numbers and semantic completions in all settings (Appendix H).
5 Subliminal learning is a form of steering vector distillation
We have shown that classical subliminal learning is a special case of steering vector distillation. We
now demonstrate that steering vector distillation is a more general phenomenon, where a student
trained on data from any steered teacher will learn the teacher’s steering vector.

5.2 Student models are distilled with steering vectors
The student model learns the steering vector across all settings by metrics 2 through 4 (see Figure 6c
and Figure 6d for activation difference and loss metrics and Appendix J for EAS). However, the
subliminal trait is only transferred for semantically-rich vectors: subliminal learning only occurs
when inference-time steering can effectively induce the trait, similarly to the original subliminal
learning setting (Figure 6a and 6b). Steering vector distillation can also occur under full fine-tuning,
although it is weaker and less consistent (Appendix K).
6 Mechanistic Explanation
We ask a natural follow-up question: how does fine-tuning install vstudent in the first place? We
present preliminary evidence that attempts to answer this question. We find that while the gradient on
teacher-generated data carries a small, consistent component along vteacher, this component is only
amplified during fine-tuning by adaptive optimizers, which plain SGD does not reproduce. Cloud
et al. [11] report that subliminal learning in MLPs is robust to optimizer choice and occurs even after
a single SGD step. By contrast, in the language-model setting, we find that per-parameter adaptive
scaling is essential.

6.2 Subliminal learning occurs under low-rank training with adaptive optimizers
We investigate the conditions under which subliminal learning arises across three models: Qwen2.5-
7B-Instruct [29], Llama-3.1-8B-Instruct [15], and OLMo-3-7B-Instruct [26]. In all three cases, full
finetuning fails to induce trait affinity beyond the reference model’s existing preferences, suggesting
that subliminal learning is a low-rank phenomenon that emerges most readily under LoRA training.
Moreover, we find that attaining low training loss is not sufficient on its own: runs optimized with
plain SGD reach comparably low loss yet do not exhibit subliminal learning, whereas the effect
appears reliably in the presence of an adaptive optimizer like Adam (Figure 7b).

of this variant that retains only the bottom 10% of per-parameter scales, keeping the gradients of
the largest parameters but replacing the rest with the geometric mean of the scales. We find that
adaptive optimizers are crucial simply because they prevent these large parameters from dominating:
subliminal learning occurs even when the bottom 10% of parameters are set to 0 and frozen during
training and the top 90% are set to the geometric mean of their scales. Concurrent work [ 24] has
found that SGD can induce subliminal learning with hyperparameter tuning, but we have not been
able to replicate this using their setting. Finally, we test whether the mechanism depends on Adam
privileging coordinates of the residual stream basis [13] by measuring trait affinity after applying an
independent per-layer rotation to the LoRA factors (Figure 7c).
7 Related Work
Subliminal Learning.Cloud et al. [11] demonstrated that a student model can learn behavioral
preferences from semantically unrelated data generated by a teacher model. Zur et al. [41] claim that
subliminal learning occurs as a result of token entanglement, where certain tokens disproportionately
9
===== PAGE 10 =====
influence other tokens. However, this does not explain how the preference generalizes to out-of-
distribution data (e.g. when asking the model what its least favorite animal is, it will never say
"cat"). Schrodi et al. [30] showed that even when excluding entangled tokens from the training data,
subliminal learning still occurred. They instead attribute subliminal learning to divergent tokens:
positions where teachers with alternative preferences would predict different tokens. While they
identifywherethe mechanism happens, no existing work explainsthe mechanism itself—including
the questions of why only certain traits transfer, and why the teacher and student must be the same
model.
Predicting and eliciting unintended generalization in LLMs.Fine-tuning language models on
narrow tasks is known to produce unintended generalization, including weird generalization [ 5],
out-of-context reasoning (OOCR) [ 4, 37], and emergent misalignment [ 6, 31]. Both OOCR and
emergent misalignment have been mechanistically linked to linear directions in activation space
[37, 31]. Wang et al. [38] developed a method for predicting which unintended biases a model will
learn from seemingly benign data. Aden-Ali et al. [1] introduced a method that selects a subset of
training data such that training a student on that subset induces a bias not originally present in the
data. Concurrent work [23] introducedsubliminal steering, a method to strengthen the transfer of
subliminal learning through training a steering vector to maximize the likelihood of the bias. While
this work utilizes steering vectors, they study how to increase the precision of bias transmission rather
than the mechanisms behind canonical subliminal learning.
Features as directions.High-level concepts in language models are widely modeled as linear
directions in activation space [7, 12, 28]. Past work has shown that these directions can be extracted
from the model [3, 21, 22] using the difference-in-mean activations from contrastive pairs of inputs
[9, 27]. Multiple techniques demonstrate that adding feature vectors to the residual stream can
allow for control over the model’s behavior [21, 22, 35, 40]. For example, recent work has isolated
directions representing concepts like "refusal" [ 2], different model personas [ 10], and emergent
misalignment [31]. Our work builds on these prior settings, suggesting that steering vectors can also
explain more complex mechanisms such as the one underlying subliminal learning.
8 Discussion
A framework for subliminal learning.Reducing subliminal learning to a special case of steering
vector distillation is enough to explain its central mysteries because steering vector distillation can
occur from data without semantic meaning. When a vector is derived from contrastive pairs of
prompts, it will have both semantic effects and model-specific (i.e. non-semantic) effects [36, 21, 22].
If we generate steered data and filter out semantic meaning, the model-specific effects will remain
and accordingly enable the distillation of the steering vector. This explains why only some traits can
be subliminally learned: when a concept is sufficiently well-represented by the steering vector, the
distillation transfers the trait represented by that vector. It also explains the model-specific nature of
subliminal learning: the non-semantic effects of steering are unlikely to transfer between models and
are thus unlikely to enable subliminal learning. Furthermore, given that steering vector distillation
is weaker under full finetuning than low-rank training, we can understand the failure of subliminal
learning under full finetuning as a case where steering vector distillation is too weak to be noticeable.
Limitations.Our analysis has several limitations. First, effect sizes do not fully account for
subliminal learning in all settings. For Qwen, adding vstudent does not completely recover the cat
affinity induced by LoRA fine-tuning, and ablating this direction does not fully eliminate the effect.
Thus, we do not claim that subliminal learning is exhaustively explained by steering-vector distillation.
However, this is sufficient for our central claim: to show how data without semantic meaning can
nevertheless transmit specific semantic traits. Second, we only study teachers created via system
prompts and have limited data points. Third, we find that per-coordinate adaptive scaling is necessary
to install vstudent, but we do not have a principled explanation for this. Fourth, we find that Gemma-3-
4b-it exhibits significantly weaker subliminal learning than Qwen2.5-7B-Instruct, suggesting that
certain reference models are less susceptible to subliminal learning. Additionally, we find subliminal
learning is most reliably elicited under low-rank training with adaptive optimizers in LLMs, whereas
this is not the case in Cloud et al. [11]’s account of subliminal learning in MLPs. Finally, our main

experiments on Qwen2.5-7B-Instruct run for 2 epochs of training, but when extending Qwen training
to 10 epochs, we observe less salient results.

filter the data, we first remove all samples that contain the target word ("cat") and then use GPT 5.4-nano as an

C.2 Subliminal learning via paraphrasing datasets
Gisler et al. [14] and Bozoukov et al. [8] recently found that subliminal learning can occur by training on a
natural language dataset that was paraphrased by a biased teacher. We replicate their work showing that a

subliminal learning is mediated by a steering vector in this setting. Similar to Bozoukov et al. [8], we create the
training data by paraphrasing the Alpaca dataset [33], instructing the teacher model to preserve semantic content.
We also use a stricter filtering process here to account for the greater risk of semantic content in natural language
data, described in Appendix C.1.

F.2 Mean-centeringv teacher
Since vteacher is derived solely from number sequences, we expect that there is significant overlap in these
directions across different traits. To strengthen the claim that vteacher is similar to its respective vstudent, we
create versions of the two vectors where we eliminate the component that is shared across traits. Thus, we
define vteacher_resid as the mean-centered version of vteacher and vstudent_resid as the mean-centered version of vstudent.
We compare the cosine similarity between vteacher_resid and vstudent_resid pairwise across all traits. We find that
corresponding teacher and student have high cosine similarity, while cross-trait teachers and students are typically
near 0. Results in Figure 14.

I Steering vector distillation experiment details
I.1 Extracting difference-of-mean steering vectors
Semantic vectors.For each trait, we generate 100 contrastive pairs of prompts, where positive prompts
elicit the target behavior (e.g. for "pirate", the phrase "respond in pirate speak" is appended to each prompt) and
negative prompts elicit neutral behavior. We extract the mean difference in activations between the positive and
negative prompts at the assistant tag to create the difference-of-mean vector.
Random vectors.We create random vectors with seeds 42, 123, and 456. For random SAEs, we randomly
select an SAE feature vector from https://www.neuronpedia.org/qwen2.5-7b-it/3-resid-post-aa with seeds 42,
123, and 456.
I.2 Training details
For each vector, we train LoRA adapters with seeds 42, 123, and 456.
Data generation.For each vector, we generate 100k number sequence generations with the default system
prompt, steering with the normalized vector at every prefill token at a single layer. For random vectors, we steer
at layer 16 with α= 40 , and for SAE feature vectors we steer at layer 20 with α= 50 . For semantic vectors, we
steer at the layer and alpha combination that yields the highest behavioral shift rate with inference time steering
on evaluation questions, which is the following:
• Pirate: layer 16,α= 100
• Shakespeare: layer 16,α= 150
• Nature: layer 20,α= 50
• Happiness: layer 16,α= 40

Mv teacher lowers loss on number completions
If vteacher truly carries the cat-system signal that drives subliminal transfer, patching it into the reference model
without any system prompt should make the teacher’s own number completionsmore probable. We test this
directly.
We score the teacher’s number-generation data under three conditions on Qwen2.5-7B-Instruct, with no system
prompt at score time (matching student-training conditions): (i) the unsteered refernce model; (ii) the reference
model with α·v teacher added at the assistant-tag position in every transformer block (α=0.5); and (iii) a roofline
that re-scores the same data with the cat system prompt actually in context. We report NLL/token.
Steering at the assistant tag lowers NLL/token from 0.7273 to 0.6823, closing roughly 32% of the baseline-to-
roofline gap (∆=−0.045 of a maximum −0.141; Figure 20). A single residual direction at a single position
thus recovers a meaningful share of what the cat system prompt does to next-token statistics on numbers —
evidence thatv teacher is the carrier of the trait signal rather than a post-hoc artifact of training. (Figure 20).

N Subliminal learning in MLPs
Cloud et al. [11] demonstrate that subliminal learning is a broader concept which has an analogous phenomenon
in MLPs. When a teacher MLP is trained to classify MNIST handwritten digits [ 20] using 10 regular output
logits, a student distilled only on three auxiliary logits from the teacher achieves over 50% accuracy on the
MNIST test set. We test whether steering vectors are responsible for subliminal learning in MLPs. Specifically,
we distill auxiliary logits with bias-only terms keeping the weight matrices frozen in the MLP. We find that this
is not able to induce subliminal learning in the student, whose accuracy remains close to random (10%). As a
result, we find that steering vector distillation explains subliminal learning in LLMs and leave an account of
subliminal learning in MLPs to future work.
